Output

Jump to: Properties Operations Events

Corresponds to an output connected to the board (digital).

WARNING: Be careful when you choose the pins! Some pins may already be in use by different modules. E.g. pin 0 and 1 are used for Serial communication. If you use the Ethernet shield, some pins are already taken: 4, 10 -> 13 (for Arduino Uno) and 4, 10, 50 -> 53 (for Arduino Mega). Etc.

Properties
name:String
Mandatory. E.g.: myNewObject.
The name of the object, which needs to be a valid C++ identifier.
pin:Number
Mandatory. E.g. 3 or A3.
The pin of the input. When choosing pins, please see the remark above (about occupied pins).
initialValue:String
Default = 0 (LOW).
Possible values:
  • 0 (LOW)
  • 1 (HIGH)
contributesToState:Boolean
Mandatory. Default = true.
If true, it means this object's state belongs to the global state of the application, which is sent periodically (e.g. to the web interface). I.e. we'll be able to use this object's state (e.g. is it on?) from within the web interface.
Operations
toggleHighLow()
Toggles the state (i.e. if was HIGH => LOW; if was LOW => HIGH).
setHigh()
Sets the state to HIGH.
setLow()
Sets the state to LOW.
Events
onValueChanged(ValueChangedEvent)
Triggered when the value of the input changes. Contains the following information:
  • previousValue:int
  • currentValue:int
If digital, then previousValue and currentValue may be HIGH or LOW.
