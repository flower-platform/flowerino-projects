Input

Jump to: Properties Operations Events

Corresponds to an input connected to the board (digital or analog).

WARNING: Be careful when you choose the pins! Some pins may already be in use by different modules. E.g. pin 0 and 1 are used for Serial communication. If you use the Ethernet shield, some pins are already taken: 4, 10 -> 13 (for Arduino Uno) and 4, 10, 50 -> 53 (for Arduino Mega). Etc.

Properties
name:String
Mandatory. E.g.: myNewObject.
The name of the object, which needs to be a valid C++ identifier.
pin:Number
Mandatory. E.g. 3 or A3.
The pin of the input. When choosing pins, please see the remark above (about occupied pins).
isAnalog:Boolean
Mandatory. Default = false. Whether this pin is analog or not. Please note that not all pins can be used as analog pins.
internalPullUp:Boolean
Mandatory. Default = false.
E.g. for a push button, if our input has a pull-up or pull-down resistor, then we use false. Else, if we don't have a resistor, we'll selecttrue (i.e. we'll use an internal pull-up). In this case, when closing the circuit (e.g. pushing the button), we'll connect the input to ground.
Note:
  • With a pull-up resistor or internal pull-up: if button is NOT pressed => state is HIGH (hence the name, "pull up"); else, if button is pressed => state is LOW.
  • With a pull-down resistor, the logic is opposite: if button is NOT pressed => state is LOW (hence the name, "pull down"); else, if button is pressed => state is HIGH.
pollInterval:String
Default = 200.
Minimum time interval (in milliseconds) between accepted state changes. Changes which occur faster than pollInterval are ignored.
contributesToState:Boolean
Mandatory. Default = true.
If true, it means this object's state belongs to the global state of the application, which is sent periodically (e.g. to the web interface). I.e. we'll be able to use this object's state (e.g. is it on?) from within the web interface.
Operations
None.
Events
onValueChanged(ValueChangedEvent)
Triggered when the value of the input changes. Contains the following information:
  • previousValue:int
  • currentValue:int
If digital, then previousValue and currentValue may be HIGH or LOW.
