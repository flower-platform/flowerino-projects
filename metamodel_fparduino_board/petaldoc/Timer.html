Timer

Jump to: Properties Operations Events

The timer allows us to execute operations after a certain delay passes.

The timer can run once, it can repeat several times, or an infinity of times, cf. repeatCount.

Properties
name:String
Mandatory. E.g.: myNewObject.
The name of the object, which needs to be a valid C++ identifier.
delay:Number
Mandatory. E.g. 1000, i.e. one second.
The delay, in milliseconds, between timer events.
repeatCount:Number
Mandatory. Default = 0.
The total number of times the timer is set to run. If the repeat count is set to 0, the timer continues indefinitely, or until the stop() operation is invoked or the program stops. If the repeat count is nonzero, the timer runs the specified number of times.
autoStart:Boolean
Mandatory. Default = true.
If true then the timer begins to run when the object is created (i.e. on program start). If false then we need to call explicitly the start() operation.
Operations
start()
Starts the timer, if it is not already running.
stop()
Stops the timer. When start() is called after stop(), the timer instance runs for the remaining number of repetitions, as set by the repeatCount property.
reset()
Stops the timer, if it is running, and sets the currentCount property back to 0, like the reset button of a stopwatch. Then, when start() is called, the timer instance runs for the specified number of repetitions, as set by the repeatCount value.
Events
onTimer(TimerEvent)
Dispatched whenever the Timer object reaches an interval specified according to the delay property.
onTimerComplete(TimerEvent)
Dispatched whenever the Timer object has completed the number of repeats set by repeatCount. If repeatCount is set to 0, this event is never dispatched, as the timer runs an infinity of times.
