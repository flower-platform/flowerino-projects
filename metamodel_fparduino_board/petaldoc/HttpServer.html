HttpServer

Jump to: Properties Operations Events

It has 3 main responsibilities:
  • Serves an HTML page with the contents generated from the User Interface Diagram.
  • Serves state updates for the connected clients. I.e. the clients periodically request state updates; when they do, the server sends a JSON containing information about all the objects that contribute to the state (i.e. their contributesToState property = true).
  • Receives commands (configured from the User Interface Diagram as well). When a command is received, an event is dispatched and operations can be invoked.
Properties
name:String
Mandatory. E.g.: myNewObject.
The name of the object, which needs to be a valid C++ identifier.
ipAddress:String
Mandatory. E.g.: 192.168.1.100.
The IP address that will be assigned to the server. It should be unique in the network. We select an IP that is compatible with our network settings (e.g. subnet mask). E.g. if our computer has an IP like: 192.168.1.4, with a subnet mask of 255.255.255.0, then we can assign something like 192.168.1.100.
port:Number
Mandatory. Initial: 80.
The server port.
macAddress:String
Mandatory. Initial: AA:BB:CC:DD:EE:FF.
Some network boards/shields have a label with the MAC address. In this case, we can use it. Otherwise, we can leave the default value (which is very probably unique in our network).
Operations
None.
Events
onCommandReceived(HttpCommandEvent)
Dispatched when the server receives a command for the specified mapping. E.g. for mapping = myCommand, when an URL like http://SERVER_URL:SERVER_PORT/myCommand (e.g. http://192.168.1.100/myCommand) is called by a client, then the corresponding event is dispatched (so the linked operations are invoked).
